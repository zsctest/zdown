# 项目规则

## 文件保护

未经用户明确授权，**禁止删除**以下文件：

- `Cargo.toml`
- `Cargo.lock`

**Why:** 这两个文件是 Rust 工作区的核心配置与依赖锁定文件，删除会破坏构建可复现性，且 `Cargo.lock` 丢失后可能引入难以排查的依赖漂移。

**How to apply:**

- 任何涉及 `rm`、`git clean -fd`、`cargo new` 覆盖、重构移动等操作前，必须检查目标路径是否命中上述文件。
- 即使用户要求"清理"或"重置"工作区，也必须先列出将受影响的文件，得到用户对 `Cargo.toml` / `Cargo.lock` 的明确许可后才能继续。
- 如需修改（而非删除）这两个文件，按正常编辑流程处理，本规则不限制修改。
