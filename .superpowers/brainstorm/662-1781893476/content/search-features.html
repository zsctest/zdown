搜索功能需要哪些能力？

Ctrl+F 搜索栏应包含哪些搜索选项和替换能力？

搜索选项

🔤 区分大小写 (Aa)

切换按钮，默认关闭。开启后 "Hello" 不会匹配 "hello"。

实现难度：低

🔤 全词匹配 (ab|)

切换按钮，默认关闭。开启后搜索 "foo" 不会匹配 "foobar"。

实现难度：低

.* 正则表达式

切换按钮，默认关闭。开启后搜索词被解释为正则表达式。

实现难度：中（需引入 regex crate）

替换能力

🔄 替换当前匹配

替换高亮的当前匹配项，然后跳到下一个匹配。

实现难度：低

🔄 替换全部

一次性替换文档中所有匹配项。支持撤销（所有替换作为一个 undo 组）。

实现难度：中

A

基础搜索（仅查找）

文本搜索 + 区分大小写 + 全词匹配。无替换功能。后续迭代再加替换。

MVP，快速可用

B

查找+替换（推荐）

文本搜索 + 区分大小写 + 全词匹配 + 替换/替换全部。正则表达式后续迭代。

一步到位，功能完整

C

全部功能

文本搜索 + 区分大小写 + 全词匹配 + 正则 + 替换/替换全部。一次性实现所有能力。

实现时间较长
