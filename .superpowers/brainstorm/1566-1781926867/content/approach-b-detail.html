方案 B：本地 mermaid-cli 详解

通过 Node.js 命令行工具在本地渲染 Mermaid 图表

工作流程

Markdown 源码                  zdown 内部流程                    输出
============                  ================                  ====
```mermaid                   1. 解析 → CodeBlock
graph TD                     2. 检测 language="mermaid"
    A → B                    3. 写入临时 .mmd 文件
    B → C                    4. 调用 mmdc -i in.mmd -o out.svg
```                          5. 读取 out.svg → 嵌入渲染       SVG 图表
                             6. 清理临时文件

依赖关系

用户系统                  zdown（Rust）                  mermaid-cli

Rust 侧实现要点

// 伪代码
fn render_mermaid_cli(content: &str) -> Result<String> {
    let tmp = tempfile::tempdir()?;
    let input = tmp.path().join("diagram.mmd");
    let output = tmp.path().join("diagram.svg");
    fs::write(&input, content)?;

    let status = Command::new("mmdc")
        .arg("-i").arg(&input)
        .arg("-o").arg(&output)
        .arg("-b").arg("transparent")  // 透明背景
        .status()?;

    if !status.success() {
        return Err("mmdc 执行失败".into());
    }
    fs::read_to_string(&output)
}

✓ 优点

  • 完整 Mermaid 语法支持（与官方同步）
  • 离线工作，不依赖外部网络
  • SVG 输出质量高，可嵌入 PDF
  • 可配置主题、尺寸、背景

✗ 缺点

  • 需要用户安装 Node.js + mmdc（~200MB）
  • 首次启动慢（启动 Chromium）
  • 进程调用有开销（每次渲染 ~1-3 秒）
  • 平台兼容性（Windows/Linux/macOS 路径处理）
  • puppeteer/Chromium 在 headless 环境可能失败
错误处理层级

1️⃣ mmdc 未安装 → 降级为代码块语法高亮 + 提示"安装 mmdc 以渲染图表"

2️⃣ Mermaid 语法错误 → 显示错误信息在图表位置

3️⃣ 超时（>10s） → 终止进程，显示超时提示

4️⃣ Chromium 启动失败 → 降级提示，建议检查环境
