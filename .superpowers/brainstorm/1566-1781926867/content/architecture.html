Mermaid 渲染架构

基于 mermaid.ink 的纯 HTTP SVG 渲染路径

核心模块：mermaid_renderer

新增一个轻量 crate crates/mermaid_renderer/，负责：

  1. 将 Mermaid 语法编码为 mermaid.ink URL
  2. HTTP GET 获取 SVG 响应
  3. 返回 SVG 字符串供各视图/导出使用

URL 编码方式

// 方式 A：GET 请求（简单图表）
https://mermaid.ink/img/pako:

// 方式 B：POST 请求（复杂图表，避免 URL 长度限制）
POST https://mermaid.ink/img
Body: 

// 推荐：优先 GET，超过 URL 长度阈值后降级 POST
调用流程
CodeBlock(lang="mermaid")
  ↓
MermaidRenderer::render(content)
  ↓
encode → pako deflate → base64url
  ↓
ureq::get("mermaid.ink/img/pako:...")
  ↓
SVG string / Error
  ↓
嵌入 egui 图像 / HTML <img> / PDF 嵌入
降级链
1️⃣ 网络请求成功 → 返回 SVG
2️⃣ 网络超时 (10s) → 降级为代码高亮
3️⃣ Mermaid 语法错误 → 显示错误文本
4️⃣ 非 mermaid 代码块 → 原样高亮
模块接口设计
/// Mermaid 渲染器
pub struct MermaidRenderer {
    client: ureq::Agent,  // 连接复用
    timeout: Duration,
    cache: LruCache, // 可选缓存
}

impl MermaidRenderer {
    /// 渲染 Mermaid 源码为 SVG 字符串
    fn render(&self, source: &str) -> Result

    /// 判断 CodeBlock 是否为 mermaid 图表
    fn is_mermaid(cb: &CodeBlock) -> bool
}
